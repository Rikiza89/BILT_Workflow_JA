アプリケーションガイド

BILT
概要

このアプリケーションは BILT を使ったオブジェクト検出のための統合ワークフローツールです。

BILT BILTモードの特徴
  • BILT (Because I Like Twice) を使用
  • ImageNet事前学習済みバックボーンで少ない画像でも学習可能
  • CPU フレンドリー設計（Raspberry Piでも動作）
  • 5つのバリアント (spark / flash / core / pro / max)
  • PyTorch ベースのカスタムアーキテクチャ
  • CIoU 損失 / EMA / モザイク拡張に対応
  • FPN アーキテクチャでマルチスケール検出
  • CUDA GPU を自動検出・使用
  • チェックポイントは float16 保存でファイルサイズ半減
クイックスタート
サービスの起動順序:
  1. 1
    python bilt_service.py — BILTバックエンド (port 5002) を起動
  2. 2
    python app.py — メインアプリ (port 5000) を起動
または python launcher.py で全サービスを一括起動できます。
ラベリング (アノテーション)

ラベリングページでは、画像にバウンディングボックスを描画してクラスラベルを付与します。BILTトレーニングには標準のYOLO形式テキストラベルが使われます。

ラベリング手順:
  1. 「プロジェクト」ページで新規プロジェクトを作成
  2. プロジェクトを開き、クラス名を設定
  3. カメラまたはファイルアップロードで画像を追加
  4. 画像を選択し、キャンバス上でボックスを描画
  5. クラスを選択してラベルを保存
  6. train / val の両方のスプリットに画像を振り分ける
キーボードショートカット:
1-9クラスを素早く選択
Delete選択したボックスを削除
Ctrl+Z元に戻す
←/→前/次の画像
Space保存
BILTによる再ラベリング: 一度でもBILTトレーニングを行った場合、その .pth モデルを使って「BILTで再ラベリング」セクションから自動ラベル付けができます。最初は手動でアノテーションしてBILTモデルを学習し、その後再ラベリングに活用してください。

ラベルファイル形式

各画像と同名の .txt ファイルを labels/ に置きます。1行1オブジェクト:

<クラスID> <x中心> <y中心> <幅> <高さ> # 例: 640×480の画像、クラス0の物体 0 0.3125 0.2917 0.3125 0.2500

全ての値は画像サイズで 正規化（0〜1）されています。

data.yaml
nc: 3 names: - cat - dog - person

nc はクラス数、names はクラス名のリストです（0番から順番に）。

必要な画像枚数の目安
クラスあたり枚数期待できる精度
2〜5枚動作はする。ノイズが多いため conf=0.05 で確認推奨
10〜50枚50エポック以上で良好な結果
50〜200枚実用的な検出器。ほとんどのユースケースに対応
200〜500枚高精度。pro や max バリアントを試す価値あり
500枚〜優秀な結果。バッチサイズとエポック数を増やすと効果的
分割の注意: 同じ画像を train と val の両方に入れないでください。val はモデルが「見たことない」画像で精度を測るためのものです。理想は80%をtrain、20%をvalに振り分けます。
BILT BILT (Because I Like Twice) とは

BILT は PyTorch ベースの軽量オブジェクト検出ライブラリです。ImageNet事前学習済みバックボーンを持ち、少ない画像でもすぐに学習を始められます。

BILTの特徴:
  • 事前学習済みバックボーン: 2〜5枚の画像からでも動く検出器を作成可能
  • CPU対応: GPUがなくても動作（小バリアントは特に高速）
  • FPN アーキテクチャ: マルチスケール特徴量で小物体にも対応
  • バックボーンウォームアップ: 最初の数エポックはバックボーンを凍結して検出ヘッドを先に収束
  • AdamW + Cosine Annealing: 安定した学習スケジューリング
  • CUDA GPU 自動検出: GPU搭載環境では自動的にGPUを使用
制限: BILTはセグメンテーション・OBBには未対応です。バウンディングボックス検出のみです。

BILT BILTバリアント

BILTには5つのモデルバリアントがあります。速度と精度のトレードオフで選択してください。

バリアント 短縮名 バックボーン 入力サイズ ファイルサイズ 用途
spark n, nano MobileNetV2 320 px ~9 MB 最高速・最小メモリ (Raspberry Piなど)
flash s, small MobileNetV3-Small 416 px ~13 MB 高速・良好な精度
core ⭐ m, medium MobileNetV3-Large 512 px ~19 MB バランス型 （初心者推奨）
pro l, large ResNet-50 640 px ~55 MB 高精度（GPU推奨）
max x, xlarge ResNet-101 640 px ~95 MB 最高精度 (GPU必須)
推奨: 初めては core から。リソースが限られる場合は spark / flash。高精度が必要で GPU がある場合は pro / max。
BILT BILTトレーニング

ワークスペースの「BILTトレーニング」パネルから実行できます。

学習の流れ:
  1. バリアントとエポック数を選択
  2. 「トレーニング開始」ボタンをクリック
  3. 損失グラフでリアルタイムに進捗確認
  4. 完了後 .pth ファイルが自動保存
出力モデルの保存先:
  • projects/<name>/runs/bilt/train_*/weights/best.pth
  • bilt_models/<name>_bilt_*.pth （検知ページから選択可）
学習時間目安 (core, CPU): 50エポック × 100枚 ≈ 30〜90分。GPU使用時は大幅に短縮されます。
BILT トレーニング パラメータ詳細 （初心者向け）

初めて使う場合は すべてデフォルト値のまま でトレーニングを開始してかまいません。

基本パラメータ
パラメータデフォルト意味・使い方
variant core 使用するモデルのサイズを選びます。小さいほど速く、大きいほど精度が上がります。
初心者は core（中型）を推奨。 Raspberry Pi など低スペック機器では spark を選んでください。
epochs 50 学習を何回繰り返すかを指定します（1エポック＝全画像を1周）。
少なすぎると「学習不足」、多すぎると「過学習」になります。画像が少ない場合（50枚以下）は 30〜50、100枚以上なら 50〜100 が目安です。
batch 4 一度に何枚の画像を使って学習を更新するかを指定します。
大きいほど学習が安定しますが、メモリを多く使います。メモリ不足になる場合は 2 に下げてください（最小値は 2）。
imgsz バリアント依存 学習・推論時に画像をリサイズするピクセルサイズです（正方形）。
空欄にするとバリアントの推奨サイズ（spark=320, flash=416, core=512, pro/max=640）が使われます。初心者は空欄推奨。
lr0 0.002 学習率（Learning Rate）の初期値です。「1回の更新でどれだけ大きく修正するか」を決めます。
大きすぎると学習が不安定になり、小さすぎると学習が遅くなります。初心者はデフォルトのままで問題ありません。
workers 0 データ読み込みに使う並列プロセス数です。
0 はメインプロセスで行います（Windows・Raspberry Piではこの設定を推奨）。マルチコアのLinux PCでは 2〜4 にすると速くなる場合があります。
device 自動 学習に使うハードウェアを指定します。空欄のとき、CUDA（GPU）が使えれば自動的にGPUを使います。
初心者は空欄推奨。 CPU強制は cpu、特定GPUは cuda:1、Apple Siliconは mps。
ウォームアップ・学習率スケジュール

バックボーンを段階的に学習させるための設定。初心者はデフォルトのまま推奨。

パラメータデフォルト意味・使い方
warmup_epochs 3 最初の数エポック、バックボーン（画像の特徴を抽出する部分）を凍結して検出ヘッドのみ学習させます。
これにより学習初期の不安定な更新でバックボーンが壊れるのを防ぎます。0 に設定するとウォームアップなしで全体を同時に学習します。画像が非常に少ない場合（20枚以下）は 0 にすると良い場合もあります。
backbone_lr_mult 0.1 ウォームアップ後、バックボーンに適用する学習率の倍率です。
0.1 の場合、バックボーンは検出ヘッドの1/10の速さで学習します。これにより既に学習済みの特徴抽出能力を壊さずに調整できます。
cos_lr_min 1e-6 コサイン学習率スケジューラが最終的に下げる学習率の最小値です。学習が進むにつれ学習率を徐々に下げ、最後の微調整を安定させます。初心者はデフォルトのまま推奨。
weight_decay 1e-4 重みの大きさにペナルティをかけて過学習を抑制する正則化パラメータです。
大きくするほど過学習しにくくなりますが、学習が遅くなります。初心者はデフォルトのまま推奨。
grad_clip 5.0 勾配クリッピングの閾値です。学習中に勾配が異常に大きくなった場合に、この値で上限を設けることで学習の発散（loss が NaN になること）を防ぎます。初心者はデフォルトのまま推奨。
損失関数パラメータ
パラメータデフォルト意味・使い方
focal_alpha 0.25 Focal Loss のクラス均衡パラメータです。
検出対象（正例）と背景（負例）の比率が偏っている場合に、少ない方のクラスに大きな重みを付けて学習させます。データに背景ばかりが多い場合は少し大きくする（例: 0.5）と改善することがあります。
focal_gamma 2.0 Focal Loss の難易度調整パラメータです。
すでに正しく検出できている「簡単なサンプル」の損失を小さくして、「難しいサンプル」の学習に集中させます。値が大きいほど難しいサンプルへの集中度が上がります（通常 1〜5 の範囲）。
box_loss_weight 1.0 バウンディングボックスの位置精度（回帰損失）の重みです。
大きくするとボックスの位置をより重視して学習します。クラスは正しく当たっているがボックスがずれる場合は大きくするとよいです。
データ拡張パラメータ

データ拡張は学習画像を自動加工して水増しする技術です。

パラメータデフォルト意味・使い方
augment true データ拡張（反転・色変化）を有効にするかどうかです。
基本的に有効のまま推奨。 無効にすると学習画像がそのままの状態のみで学習され、過学習しやすくなります。
flip_prob 0.5 学習画像を左右反転させる確率です。
0.5 は「50%の確率で反転させる」という意味です。左右非対称な物体（例: 文字、特定の方向を持つ製品）を検出する場合は 0.0 に設定してください。
color_jitter_b 0.4 明るさ（Brightness）をランダムに変化させる強度です。
0.4 は「最大±40%の明るさ変化を加える」という意味です。照明条件が大きく変わる環境では値を大きく（0.6〜0.8）してください。
color_jitter_c 0.4 コントラスト（Contrast）をランダムに変化させる強度です。
明暗の差がばらつく環境（屋外など）では大きくすると効果的です。
color_jitter_s 0.4 彩度（Saturation）をランダムに変化させる強度です。
色の鮮やかさが変わる環境に有効です。グレースケールに近い画像を扱う場合は 0.0 にしても良いです。
color_jitter_h 0.1 色相（Hue）をランダムに変化させる強度です。
0.1 は「最大±10%の色相変化」です。色の分類が重要なタスク（例: 赤い物体のみを検出）では 0.0 にしてください。
まず試すべき調整順序:
  1. デフォルト値のままトレーニング実行
  2. val_lossが下がらない → lr0 を 0.001 に下げる
  3. ボックスがずれる → box_loss_weight を 2.0 に上げる
  4. 左右非対称な物体 → flip_prob を 0.0 に設定
  5. 照明変化が激しい → color_jitter_b/c を 0.6 に上げる
BILT 高度なトレーニングオプション

以下の機能は精度向上に効果的ですが、初心者は基本パラメータに慣れてから試してください。特に pro/max バリアントで効果が顕著です。

パラメータデフォルト意味・使い方
lr_warmup_epochs 0 LR線形ウォームアップのエポック数。大きいデータセットや pro/max では 5 程度推奨。
use_ciou false CIoU回帰損失を使用。true にするとボックス精度が向上。pro/max では有効化推奨。
use_ema false 指数移動平均を有効化。汎化性能向上に効果あり。pro/max では有効化推奨。
ema_decay 0.9999 EMA減衰率の上限。データセットサイズに自動調整されるためデフォルトのまま推奨。
mosaic false 4画像モザイク拡張を有効化。小さいデータセットでの過学習抑制に効果的。
mosaic_prob 0.5 モザイク適用確率。mosaic=true の場合のみ有効。
cache_images false 学習画像をRAMにキャッシュ。高速化に効果的だがRAMが十分にある場合のみ（目安：画像枚数×1MB以上の空きRAM）。
MobileNet (spark/flash/core) 推奨設定:
lr_warmup_epochs=3, use_ciou=True, use_ema=True, mosaic=True
ResNet (pro/max) 推奨設定:
lr_warmup_epochs=5, use_ciou=True, use_ema=True, mosaic=True, cache_images=True
物体検知
BILT BILT検知の手順
  1. ワークフローページへ移動
  2. .pth モデルを選択（事前にトレーニング要）
  3. カメラを選択して検知開始
  4. conf / iou / 最大検出数を必要に応じて調整
検知パラメータ詳細:
設定デフォルト意味調整ガイド
conf（信頼度閾値） 0.80 この確率以上の検出のみ表示 検出漏れが多い → 0.50〜0.60 に下げる。誤検出が多い → 0.85+ に上げる
iou（NMS閾値） 0.45 重複ボックス除去の厳しさ 物体が密集している → 0.3〜0.5 を試す
最大検出数 5 フレームあたりの最大表示検出数 多数の物体を同時検出したい → 増やす
BILT検知には事前にBILTトレーニングで .pth ファイルを生成する必要があります。検知を停止して再開した場合、カメラは自動的に再初期化されます。
ワークフロー

ワークフローエディタでは、複数のカメラとBILT検知ステップを視覚的に組み合わせた自動化パイプラインを構築できます。

ノードタイプ:
start bilt_detection loop alert wait end

ノードをドラッグ＆ドロップで配置し、矢印でつなぐことでパイプラインを構築します。

よくある質問

  • 独自の特定クラスをゼロから学習したい場合
  • オフライン・エアギャップ環境で使う場合
  • CPU のみの環境 (Raspberry Pi など) で動かす場合
  • 軽量でカスタマイズ可能なモデルが必要な場合

ImageNet事前学習済みバックボーンのおかげで、理論上は 2〜5枚 からでも動く検出器が作れます。ただし安定した精度を得るには、クラスごとに 20〜50枚 の学習画像と 5〜15枚の検証画像を推奨します。

python bilt_service.py が起動しているか確認してください。また、bilt パッケージが Python パスに存在するか確認してください（リポジトリは ../bilt/ に配置）。

BILTモデルは .pth 形式（float16保存）で保存されます。チェックポイントには重みだけでなく、バリアント名・クラス名・入力サイズも含まれます。

保存先:
① projects/<name>/runs/bilt/train_*/weights/best.pth
② bilt_models/<name>_bilt_<variant>_*.pth（検知ページで選択）

BatchNorm の制約により最低 2 が必要です。1 に設定した場合は自動的に 2 に修正されます。

以下を順に確認してください:
  1. アノテーションを確認: ボックスが物体をしっかり囲んでいるか、クラスラベルが正しいか確認。
  2. val画像が少ない: 各クラス最低 5〜10枚 の検証画像が必要です。
  3. 学習率を下げる: lr0 を 0.001 または 0.0005 に。
  4. エポック数を増やす: epochs を 100 以上に。
  5. バリアントを小さくする: データが少ない場合は spark や flash の方が安定することがあります。

NaN は学習率が大きすぎるか、データに問題がある場合に発生します:
  • 学習率を下げる: lr0 を 0.0005 などに。
  • grad_clipを下げる: grad_clip を 1.0 に。
  • アノテーション確認: 幅または高さが0のボックスがないか確認。
  • 画像ファイル確認: 壊れた画像や極端に小さい画像が混じっていないか確認。

設定意味目安
conf この確率以上の検出のみ表示 0.5〜0.7 から試す。BILTデフォルト 0.80。

  1. まず 手動でアノテーション した画像でBILTモデルをトレーニング（最低20〜50枚）。

  3. .pth モデルを選択し、対象スプリットを選ぶ。
  4. モード（全画像/ラベル済みのみ/未ラベルのみ）を選択して実行。
  5. 自動生成されたラベルは必ず手動で確認・修正してください。
